# Workflow

## Standard Flow

1. User describes feature
2. Router selects Architect (if structural)
3. Architect proposes design
4. Threat-model analysis
5. Builder implements
6. Tester writes tests
7. Reviewer reviews
8. Summary generates changelog

## Bug Flow

1. Builder fixes
2. Tester adds regression test
3. Reviewer validates

## Rule
- No direct implementation without architecture approval if structural.
- No security-sensitive feature may bypass threat-model analysis.
