# Documents

Generated documentation artifacts from workflow agents.

## Structure
- 'documents/architect/': Architecture documentation generated by Architect including threat model.
- 'documents/builder/': Implementation documentation generated by Builder.
- 'documents/tester/': Test documentation generated by Tester.

## Naming Convention
- 'YYYY-MM-DD-<slug>.md'

## Required Content
- Architect docs: architecture summary, component responsibilities, connectivity matrix (including protocol/port where known), security controls, cost summary/assumptions, diagram paths, Threat-Model-Ref.
- Builder docs: change summary, touched files, config/migration notes, Threat-Model-Ref, known follow-ups.
- Tester docs: test scope, cases added/updated, execution evidence, coverage notes/gaps, Threat-Model-Ref coverage notes.
